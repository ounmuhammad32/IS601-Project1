What is Pytest?

️ ️

Pytest is the framework that makes it easy to write, test, and scale to support complex testing for the applications and libraries. It is the most popular Python package for testing. The basis for a rich ecosystem of testing is plugins and extensions. The way pytest is designed is as a very extensible system, easy to write plugins and there are a lot of plugins present in the pytest that are used for various purposes. Testing is very important before delivering the code in production.

Features Of pytest
  • Does not require API to use.
  • Can be used to run doc tests and unit tests.
  • Gives useful failuråe information without the use of debuggers.
  • Can be written as a function or method.
  • Has useful plugins.
    • Advantages Of pytest
  • It is open-source.
  • It can skip tests and automatically detect the tests.
  • Tests are run parallel.
  • Specific tests and subsets of tests can be run from the program.
  • It is easy to start with as it has a very easy syntax.

    How to run Pytest?

    There are few way to run pytest. First of all it should be installed on your machine. This can be done by running command

    • pip install -U pytest

    Now you can check what version of pytest is installed and if it is compatible with your version of Python by running a command

    • $ pytest --version

    Now we can proceed to run a test by running a command

    • python -m pytest

    Another way to run pytest when using Pycharm is to use play button in top bar on the right-hand side. the options are to run a test, to debug or run a test with coverage which I will soon discuss. Figure 2 shows those buttons.

    In our case however, the git we are working with to build this site as well as calculator program includes docker container that includes all necessary thing we need for the purpose of this exercise. That means we can skip and install and as long as our docker container is running, and we are using an interpreter from that container we can simply use this command

    • pytest

    What is Coverage?

    Coverage is a tool for measuring code coverage of Python programs. It monitors your program, noting which parts of the code have been executed, then analyzes the source to identify code that could have been executed but was not. Coverage measurement is typically used to gauge the effectiveness of tests. It can show which parts of your code are being exercised by tests, and which are not.

    Capabilities

    Coverage can do a number of things:

    • 1. By default it will measure line (statement) coverage.
    • 2. It can also measure branch coverage.
    • 3. It can tell you what tests ran which lines.
    • 4. It can produce reports in a number of formats: text, HTML, XML, LCOV, and JSON.
    • 5. For advanced uses, there’s an API, and the result data is available in a SQLite database.
    How to use coverage

    To start install coverage by using command

    • pip install coverage

    Next we can run coverage with command

    • coverage run or python -m coverage

    Or if using pytest plugin (this project)

    • pytest -cov or pytest --cov

    to run coverage and generate general report in terminal screen (Figure 6)

    To get more detailed report we can run

    • coverage html

    Which will generate folder with html files for each module (Figures 7 and 8) which can be opened in local host as shown.

    Both screenshots, Fig 8 and Fig 9 were taken before corrections were made hence different output than Fig 7.

    We should always aim for 100% coverage, however often it is not necessary and may consume a lot of time and resources. Not achieving 100% does not mean code is not good. Critical missed lines should always be addressed. Good judgment on what to put time into is critical in this test.

    What is Unittest?

    Unit testing is a software testing method by which individual units of source code are put under various tests to determine whether they are fit for use (Source). It determines and ascertains the quality of your code.

    Generally, when the development process is complete, the developer codes criteria, or the results that are known to be potentially practical and useful, into the test script to verify a particular unit's correctness. During test case execution, various frameworks log tests that fail any criterion and report them in a summary. The developers are expected to write automated test scripts, which ensures that each and every section or a unit meets its design and behaves as expected. Though writing manual tests for your code is definitely a tedious and time-consuming task, Python's built-in unit testing framework has made life a lot easier. The unit test framework in Python is called unittest, which comes packaged with Python. Unit testing makes your code future proof since you anticipate the cases where your code could potentially fail or produce a bug. Though you cannot predict all of the cases, you still address most of them.

    Implementing Pylint and Glossary

    Pylint Manual

    Pylint is a tool that checks for errors in your Python code. It analyzes your code and compares it to the coding standard. If you are coding in Python, the computer does not catch bugs for you. Therefore, you need a lint tool, like Pylint, to find errors.

    The default coding style used by Pylint is similar to PEP 8.

    Installation:

    Assuming that you are using Python, Pylint should be easily installable using pip. Install Pylint by running this command:
    For more information, please visit the Installation Guide

    pip install pylint

    Check where Pylint was installed by running:

    $ which pylint
    /usr/local/bin/pylint

    Running Pylint:

    Pylint is meant to be called from the command line. You can type:
    For more information, please visit the Running Pylint Guide

    pylint fileName.py

    Or you can call the directory, followed by the file. This will only work if the directory is a python package and has an __init__.py file.

    pylint directory/fileName.py

    Command Line Options:

    Pylint has many checkers. You can disable a specific checker and its messages by specifying:

    --disable=< symbol >.

    If you want to enable only some checkers or message symbols, first use

    --disable=all .

    Then,

    --enable=< symbol >.

    where, < symbol > is in a comma-separated list of checker names and message symbols.

    It is possible to disable all messages with --disable=all. This is useful to enable only a few checkers or few messages by first disabling everything and then re-enabling what you need.

    Python will search for a configuration file named pylintrc or .pylintrc

    Open in PyCharm:

    In Pycharm,
    1. Go to File/Settings/Tools/External Tools/,
    2. Click Add "+" button
    3. Fill in the arguments:
    Name: pylint
    Program: C:\\Python27\Scripts\pylint
    Parameters: etc...
    Working directory : $FileDir$
    5. Click "OK" in the Settings. Click right on the file or code
    6. Select External Tools/pylint
    7. Pylint should run inside Pycharm
    For more information and questions, please visit How to run Pylint

    Glossary:

    Attribute: attr() , creates attributes on class instances inside methods.

    Class Method: means that it is bound to the class and not the object of the class. This is useful for when we need to access the class itself and not the object. Class method can change a class state that would apply across all the instances of a class.

    Class: is a code template for creating objects. In Python, a class is created by using the keyword "class".

    Constructor: is a special kind of method that Python calls when it instantiates an object using the definitions found in your class. Python relies on the constructor to perform tasks such as assigning values to variables that the object will use.

    Design Patterns: provide a repeatable solution to a commonly occurring problem in software design. It is an essential part of object-oriented software development.

    Façade: is a structural design pattern that provides a simplified interface to a complex system of classes. It is often used in object-oriented programming.

    Factory: is an object for creating other objects

    Fixture: defines the steps and data that constitutes the arrangement phase of a test. In pytest, fixtures are functions used to define a test’s act phase.

    Instantiation: is creating a copy of the class which inherits all class variables and methods. To instantiate a class, we have to call the class as if it were a function, passing arguments that the __init__ method defines.

    Method: is a function that “belongs” to an object. For example, list objects have methods called append, insert, remove, sort , etc. In Python, you can create a list , list =[10,20,30] and apply a method such as my_list.append(40). This will add the number 40 to the existing list.

    Namespace: is a dictionary in which the keys are the object names and the values are the objects themselves. Python maintains a namespace in the form of a Python dictionary.

    Object: is the collection of various data and functions. Everything in Python is treated as an object (variables, functions, lists, tuples, dictionaries, etc). It is important to note that every object belongs to its class.

    Property: is a function in Python that returns the object of the property class and is used to create a property of the class.

    Static Method: means that it belongs to the class rather than the class’s object. A static method can be called without creating an object for the class. We can directly call the static method by referencing the class name.

    Type Cast: is a method to convert one variable data type into another data type. This can be useful when you want to add two numbers in which one existing value of a variable is an integer and the second is a string.

    Type Hint: is a formal solution to indicate the type of value within your Python code. Type hints help you maintain cleaner code and improve linters.

    Unit Test : is the first level of software testing. This guarantees that some change in the code won’t affect the code that is running without errors.
